## 2일차
- scanf, scanf_s 차이점 : int, double 등 크기가 고정인 경우는 상관없음
- scanf_s : 배열은 사용자 지정 크기이므로 3번째 매개변수로 sizeof를 사용해서 크기를 명시해줘야함
	```C
	scanf_s("%s", str, (unsigned int)(sizeof(strName)));
	```
	scanf는 입력 버퍼에 있는 문자를 가져와서 메모리에 저장
	> 엔터키가 출력버퍼에 남아있음
		- scanf 여러 번 사용 시 getchar() 함수 사용하여 개행문자 제거 
	printf는 출력 버퍼에 있는 문자를 가져와서 출력
	

- 연산자 : 컴퓨터는 '0' 제외 값 모두 참(대표적으로 1 사용)
	- 논리 연산자	
		- 논리 합 : A && B : AND연산
		- 논리 곱 :  A || B : OR연산
		- 부정 : !
	- 복합 대입 연산자 : <=, >= (대입연산자가 뒤)
	- 비트연산자 
		- &, |, ^, ~, <<, >>
		- ~ : 1은 0으로 0은 1로 : ~30 : 0001 1110 -> -31 : 1111 1110 0001
			8bit 중 MSB(최상위비트)는 부호bit - 0이면 양수, 1이면 음수 
			ex) 31은 MSB가 1이므로 나머지 상위비트를 1로 만듬
		- 좌측 시프트 시 LSB는 0으로 채움
		- 우측 시프트 시 MSB는 부호비트 값 따라 채워짐(양수 0, 음수 1)
	- 증감연산자
		- ++, --
- 선택 제어문(1. if, 2. switch) 
- 반복 제어문(1. for, 2. while)
- 선택 제어문, 반복 제어문 둘다 바꿔서 만들 수 있음

## 3일차
- 배열 : 동일한 자료형을 연속적으로 저장하기 위함
	- 장점 : 순회, 탐색이 좋음 
	- 단점 : 수정이 불편함, 삽입 시 : 더 큰 배열 생성 > 삽입 후 앞뒤로 붙여 넣어야함
- 자료구조 : 효율적으로 데이터를 가공하기 위해 사용
	- 인덱스 : 동일한 자료 연속적으로 저장되어 있어 [ ]접근 
	- 연결리스트 : 각각의 배열을 포인터로 주소를 연결 : 순회는 어렵지만 수정은 좋음
	- 배열은 인덱스 0부터 채워줘야함(삭제 시 앞으로 땡겨줘야함)
	- char 타입 배열 사용시 +1 크기로 생성(마지막 널문자('\0' 포함)
	- 문자 하나 > ' ' 사용, 문자열 > " " 사용

- 포인터 : 주소를 가리킴, 포인터 변수 > 주소를 저장할 수 있는 변수
	```c
	printf("num의 주소 : %p\n", &num);			// 시작주소
	printf("num의 값 : %d\n", num);
	int *p = &num;						// 포인터 변수라는 표시
	printf("포인터 변수 p에 저장된 값(주소 : %p\n", p);
	printf("포인터 변수 P가 가리키는 곳의 값: %d\n", *p);	// 간접참조 연산자
	num += 1;						// 포인터가 가리키고 있는 곳의 값을 +1
	printf("num : %d, *p: %d\n", num, *p);			// 가리키는 값과 포인터 모두 1증가
	*p += 1;						// 포인터 값 +1 == 포인터가 가리키고 있는 곳 값 +1
	printf("num : %d, *p: %d\n", num, *p);			// 가리키는 값과 포인터 모두 1증가

	```
	- 포인트 변수의 크기는 주소 크기(컴퓨터 비트)와 같다

	```c
	const int num = 10;
	//num = 20;

	int num2 = 10;
	num2 = 20;
	int* pnum2 = &num2;			// 간접참조
	*pnum2 = 30;				// 포인터변수 pnum2가 가리키고 있는 값을 변경
	printf("%d\n", num2);
						// 데이터 상수
	const int* pn2 = &num2;			// const가 포인터 변수 앞 : 포인터 변수를 통한 값 변경 막음
	//*pn2 = 100;
	num2 = 0;
	pn2 = &num2;

	int num3 = 40;				// 포인터 상수
	int* const pnum3 = &num3;		// 포인터 변수가 가리키고 있는 다른 주소 값 변경 막음
	*pnum3 = 100;				 
	printf("%d\n", num3);
	//pnum3 = &num2;

	const int* const pn5 = &num2; // 데이터 상수 + 포인터 상수
	//*pn5 = 1000;
	//pn5 = num3;
	num2 = 40;
	```

## 4일차
- 문자열 : 문자열이 시작되는 시작주소
	- 문자열 사용시 ""로 묶으면 데이터영역에 저장
	- 문자열 함수 
		- `gets`, `fgets` : 문자열 입력 함수(공백 입력 가능)
		- stdin : 표준 입력 버퍼(입력 스트림)
		- `puts`, `fputs` : 문자열 출력 함수 
		- stdout : 표준 출력 버퍼(스트림)
		- `strlen` : 문자열 길이 계산
		- `strcpy`, strncpy : 문자열 복사, strcpy(str1, str2) > str1(원본)에 str2(복사본)을 복사 
		- `strcmp` - 문자열 비교 함수(사전순) > strcmp(str1, str2) > str1 우선 검색시 -1반환, str2 검색시 1반환, 같으면 0 반환
		- `strcat`, `strncat` - 문자열을 붙히는 함수
		```c
		char str[100] = "Orange";
		char str2[100] = "apple";
		/* gets, fgets */
		printf("공백을 포함한 입력 >> ");
		gets(str);						// gets(입력) gets함수는 입력 필요
		fgets(str, sizeof(str), stdin); // 저장할 버퍼 크기, stdin : 표준입력버퍼, 파일입출력 가능
		/* puts, fputs */
		printf("%s\n", str);			// 서식문자에 따라 여러 자료형 출력가능, 무거움
		puts(str);						// 문자열 전용, 가벼움, 자동개행
		fputs(str, stdout);				// fputs는 크기 작성 미필요, 개행안됨
		/* strlen */
		printf("str 문자열 길이: %d", strlen(str)); // 널문자 카운트X
		/* strcpy */
		strcpy(str, str2);				// strcpy(원본, 복사) > str에 str2 복사 
		strncpy(str, "abcd", 3);		// 시작주소부터 복사할 문자갯수를 설정
		/* strcmp */
		if (strcmp(str, str2) > 0)		// a가 o보다 사전순으로 빠름
			printf("str: %s\n", str);
		/* strcat */
		strcat(str, "banana");
		strncat(str, "ORANGE", 3);		// 시작주소부터 연결할 문자갯수를 설정
		```
